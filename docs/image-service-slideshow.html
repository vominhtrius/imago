MVP Scope

Core Features

Upload

  • Direct to S3 — client gets a presigned URL, uploads directly. No server bottleneck.
  • Server-side — POST binary through our service. EXIF stripped before storage.
  • Magic byte validation — reject forged content types before decode.
  • Input size cap — 20 MB / 50 MP limit prevents image bomb attacks.

Transform

  • Resize — fit, cover, force. Passthrough when source is already small.
  • Crop — gravity-based (centre, top, smart). Smart crop keeps faces in frame.
  • Convert — JPEG ↔ WebP ↔ PNG. Explicit ?format=webp param.
  • Quality control — per-format quality, metadata strip on output.
Optional (add incrementally): Watermark · Blur · Pipeline · AVIF · Filters · Shape mask · Rotation
Design Principle

Synchronous On-Demand Processing

✓ Our approach
GET /users/123/photo.jpg?w=512 → image bytes in < 300ms ✓ One request. One response. No state. No polling.
✗ Async approach
POST /jobs/resize → { job_id: "abc" } GET /jobs/abc → { status: "pending" } GET /jobs/abc → { status: "pending" } GET /jobs/abc → { status: "done" } GET result_url → image bytes
  • <img src="..."> tags cannot poll — the browser expects bytes in one response
  • A feed scroll gives ~300ms before blank tiles are noticeable
  • On-demand + CDN = each unique transform computed once, then free forever

Key Architectural Advantage

Drogon IO threads → never call libvips → co_await pool.submit(...) → suspends without blocking ↓ VipsWorkerPool threads → all libvips runs here → VIPS_CONCURRENCY=1 per worker → parallelism at thread level

imgproxy blocks an OS thread per request.
weserv blocks the entire nginx event loop.
imago never blocks — IO and CPU are fully decoupled.

Caching Design

Each Unique Transform Computed Once. Ever.

Cache Layers

LayerTTLMechanism
CloudFront7 daysCache-Control header
ETag revalidationIndefinite304 Not Modified
Client browser7 daysInherits CloudFront

Cache Key

/users/123/photo_v2.jpg?w=512&h=512 &format=webp&quality=85 Each unique transform = separate entry No Vary header — format is explicit

Cache Invalidation — Versioned Keys

// User re-uploads photo Old key: users/123/photo_v2.jpg New key: users/123/photo_v3.jpg App stores new key → serves new URL Old URLs expire naturally after 7 days ✓ Zero CloudFront invalidation calls ✓ Zero cost ✓ Zero propagation delay
CloudFront invalidation API costs $0.005/path with ~15s delay.
Versioned keys eliminate both entirely.
API Design

Simple, Predictable, CDN-Friendly

Serve URL

https://images.yourdomain.com /users/123/photo_v2.jpg ?w=512 &h=512 &fit=cover &format=webp &quality=85 &crop=smart

Upload Endpoints

POST /v1/upload/presign → { presigned_url, s3_key } POST /v1/upload/confirm → { s3_key, ready: true } POST /v1/upload/binary → { s3_key, size_bytes } ← EXIF stripped

Why c7g Graviton3 (ARM)?

  • libvips uses ARM NEON SIMD for JPEG DCT, WebP, convolution
  • 20–30% better image throughput per dollar vs x86
  • Lower cost than c6i at equivalent performance

Deployment

  • Ubuntu 24.04 arm64 · libvips from apt · jemalloc via LD_PRELOAD
  • Image stored in ECR
  • Config via environment variables — no config files
  • CI/CD: build → test → ECR push → ECS rolling deploy
